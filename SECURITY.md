# Security Policy

## Reporting a Vulnerability

If you believe you've found a security vulnerability in Fil One, please report it privately via [GitHub Security Advisories](https://github.com/fil-one/fil-one/security) or email **security@fil.one**.

Fil One does not operate a bug bounty program and does not authorize active security testing against our infrastructure. If you encounter a vulnerability during normal use of the product, we welcome a responsible report through the channels above.

## Security contacts

For questions about this policy, contact the [Filecoin Foundation security team](https://fil.org/security).
